# 6. 함수명 네이밍 규칙

함수 이름은 `camelCase`로 작성하며, **동사**나 **부사**로 시작하여 어떤 작업을 하는지 혹은 어떤 상황에서 호출되는지 명확히 드러나도록 작명합니다.

## 가이드라인
- **동사로 시작하는 경우**: 해당 함수가 수행할 액션(가져오기, 저장하기, 생성하기, 계산하기 등)을 가장 첫 단어로 사용합니다.
- **부사로 시작하는 경우**: Event Handler (예: 클릭했을 때, 제출했을 때 등)는 보통 `on`, `handle` 뒤에 동작 시점을 명시하여 작명합니다.

```javascript
// ✅ Good

/** 유저 정보를 가져옵니다. */
function fetchUserInfo() {
  // ...
}

/** 폼이 제출되었을 때 호출됩니다. */
function handleFormSubmit() {
  // ...
}

/** 유효성 검사를 수행합니다. */
function validateUserInput() {
  // ...
}
```

```javascript
// ❌ Bad

// 명사로만 조합되어 동사(역할)이 보이지 않음
function userPassword() {
  // ... // 비밀번호를 가져오는건지 설정하는건지 알 수 없음
}

// 모호한 액션 (어떤 동작을 하는지 한눈에 파악 힘듦)
function doAuth() {
  // ... // auth 체크인지 로그인인지 로그아웃인지 알 수 없음
}
```
